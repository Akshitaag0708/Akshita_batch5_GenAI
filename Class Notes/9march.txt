types of method:
->class method:
called through object as well as class name . any u can use 
can be accessed via class only
They are defined using the @classmethod decorator and take the class itself as the first parameter
useful for tasks that involve the class rather than the instance, such as creating class-specific behaviors or modifying class-level attributes.
first argument will be cls according to industrial standard rule

->object method :
A method defined within a class, taking `self` as the first parameter, representing the instance.
This allows the method to access and manipulate the instance's attributes.
can access both object and class properties

note : difference between class method and object method is if u want to change class property via object method then it will be changed for that object only but if you change it via class method it will be changed for all object

->static method:
not bound to either the class or its instances. 
They are defined using the @staticmethod decorator and do not take a reference to the instance or the class as their first parameter.
useful for tasks that do not depend on instance-specific or class-specific data.
Cannot access or modify class or instance attributes.
