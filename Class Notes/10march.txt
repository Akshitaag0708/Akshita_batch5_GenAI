Inheritance:
when one class(derieved/child/subclass) derieve properties or methods of another class(base/parent/superclass)
3 types in python:
-> single : a child class inherits from just one parent class
-> multiple:  child class can inherit from more than one parent class
-> multi-layer:  a class is derived from another derived class (like a chain)

super()function
super() function is used to call the parent class’s methods. 
In particular, it is commonly used in the child class’s __init__() method to initialize inherited attributes. 
This way, the child class can leverage the functionality of the parent class

@property decorator:
helpful in defining the properties effortlessly without manually calling the inbuilt function property().
treats the function as attribute 
here until and unless we are calling the method the value of propert remains the previous one only
what if i want to create a propert whose value is dependable upon some operation and if any value of operation if changes , the property value also get updated
in these types of cases we will use @property decorator

polymorphism:
ability of an entity (like a function or object) to perform different actions based on the context.
->compile time : method overriding 
-> runtime : method overriding

operations:
3 major operation/modes:
read(r) , write(w) , append(a)
for each and every mode there are 4  categories
in write:
->only write(w)
->write+read(w+)
->write binary(wb)
->write and read binaries(wb+)

in read mode:
->only read(r)
->read+write(r+)
->read binary(rb)
->read and write binaries(rb+)

in append mode:
->only append(a)
->append+read(a+)
->append binary(ab)
->append and read binaries(ab+)


for write operation 2 functions:
-> write(str_data):single line of data
-> writelines([line1,line2,......,..]):multiple lines of data

note: if file is not present then it will create one then perform write operation
note: if file is already present ,then it will override previous data with new one

note - it is not necessary to close the file
